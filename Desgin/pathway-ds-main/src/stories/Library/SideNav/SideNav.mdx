---

A persistent vertical navigation panel for Ministry Brands Amplify. Every module gets its own `SideNav` instance — it renders the primary navigation tree and, at all times, communicates where the user is in it. This is module-level navigation, not global app navigation and not action buttons.

SideNav supports **two levels of depth** (Level 0 parents, Level 1 child-destinations), **two sidebar widths** (240 px expanded, 72 px collapsed), and **three responsive modes** (push on desktop, overlay on tablet, hidden-overlay on mobile).

When the menu is taller than the panel, it scrolls via the system **[Scrollbar component](?path=/docs/library-scrollbar--docs)** (`<Scrollable>`) — the native bar is hidden and a slim overlay thumb is drawn, so scrolling looks identical on every OS and never shifts the 240 px / 72 px rail. SideNav nests it rather than rolling its own scrollbar.

## Anatomy

The SideNav is made up of a container, a stack of items, and a collapse control at the bottom. Items can be Level 0 (with an icon) or Level 1 (no icon, indented). Every item has an indicator-stripe column on the left — 4 px wide, structural, always present in the DOM. The stripe only paints when the item is Active or Trail-collapsed.

```
SideNav.Container                       240 px expanded · 72 px collapsed
└── SideNavMenu                         flex column · gap 6 px · Gap/XTight
│   ├── SideNavItem · Level 0           48 px min-height
│   │   [indicator 4 px][leading icon 24×24][label flex][row-end 40×24 · groupers only]
│   ├── SideNavItem · Level 0 (grouper expanded)
│   │   ├── SideNavItem · Level 1       no icon · padding-left 32 px
│   │   └── SideNavItem · Level 1
│   └── SideNavItem · Level 0
└── Collapse_Expand_Nav_Container
    ├── Divider                         1 px · Stroke/Static/Neutral/Light
    └── CollapseButton                  item-like row, no indicator stripe
```

Level 1 children are always destinations — they never group further. This is a hard constraint, not a design convention: the data layer enforces it.

## States

Every item can be in one of five states. Fill, text colour, icon colour, and the indicator stripe change together — never mix tokens from different states.

<Canvas of={Stories.StateMatrix} />

**Reading the matrix.** The first three states (Base, Hover, Active) are about what *this* item is doing. The last two are about *this grouper's children*:

- **Trail — expanded** applies to any Level 0 grouper whose children are visible, regardless of which child (if any) is active. The fill is muted (`Fill/NavItem/Trail`, ~2% black), the text is active-coloured, the icon stays grey.
- **Trail — collapsed** applies when a grouper's active child is hidden — either because the grouper itself is closed, or because the whole sidebar is at 72 px. It's visually identical to Active; the only difference is semantic (Active is a leaf, Trail-collapsed is a parent).

Trail-collapsed means "something inside me is selected, and you can't see it from here." That's why it shares Active's styling — the user needs to see that the grouper itself is on the active path.

## Token mappings

The SideNav binds every colour to a semantic variable in the `Contextual/NavItem/*` family, plus `Fill/Static/Info/Subtle` for the divider and border. No raw hex, no primitives.

### Fill

<Canvas of={Stories.TokensFill} />

Note that `NavItem/Hover` and `NavItem/Trail` are different tokens even though they look similar (4% vs 2% black). They've diverged over time; if they ever fully merge, it'll happen in the token library, not in the component.

### Text

<Canvas of={Stories.TokensText} />

There's no separate "trail text" token — every trail state reuses `Text/NavItem/Active`. If you need the active ink, it's `#1b2d57`.

### Icon

<Canvas of={Stories.TokensIcon} />

The Active icon token (`#2d4889`) is also the colour of the **indicator stripe**. Same variable, two places. When Figma updates this token, both repaint together.

### Section Label

The **SectionLabel** (Figma node 40006794-5977) is the header row inside the flyout popover shown when hovering a collapsed grouper. It uses a distinct type style — small semibold uppercase — separate from the nav item label style.

The 72 px rail is a desktop-and-tablet pattern. On mobile (< 768 px) the component is fully hidden by default; hover popovers don't apply to touch-only screens, and 72 px would consume ~20% of a 390 px viewport. The TopNav hamburger is the sole toggle there.

## Accessibility

SideNav uses the ARIA **tree** pattern — `role="tree"` on the list, `role="treeitem"` on each item, `aria-expanded` on groupers only, `aria-current="page"` on the active destination. The entire tree is a single tab stop; arrow keys move within it.

<table>
  <thead>
    <tr><th style={{width:"180px"}}>Concern</th><th>Implementation</th></tr>
  </thead>
  <tbody>
    <tr><td>Tree landmark</td><td><code>{'<ul role="tree" aria-label="Main navigation">'}</code></td></tr>
    <tr><td>Active destination</td><td><code>aria-current="page"</code></td></tr>
    <tr><td>Grouper state</td><td><code>aria-expanded="true|false"</code> on groupers only — never on leaves</td></tr>
    <tr><td>Keyboard</td><td>Tab to enter · ↑/↓ to move · ←/→ to collapse/expand or descend · Enter/Space to activate</td></tr>
    <tr><td>Collapsed rail</td><td>Icon-only items need <code>aria-label</code> (or a visually-hidden span) since labels are hidden</td></tr>
    <tr><td>Touch target</td><td><code>min-height: 48px</code> on every item</td></tr>
    <tr><td>Focus ring</td><td><code>outline: 2px solid #3555a0; outline-offset: 2px</code> (uses Icon/NavItem/Active)</td></tr>
    <tr><td>Reduced motion</td><td>The width transition (<code>--motion-duration-6</code>) and popover animations collapse to a simple opacity fade</td></tr>
  </tbody>
</table>

The current reference demo uses `role="button"` on `<div>` items as a visual scaffolding baseline — **not** production-ready. Production code needs native `<button>` / `<a>` elements, roving-tabindex focus management, and the full arrow-key handler set. See `sidenav-spec.md` §13.3 for the complete keyboard table.
