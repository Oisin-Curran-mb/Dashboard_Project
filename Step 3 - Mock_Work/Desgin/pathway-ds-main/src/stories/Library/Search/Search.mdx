The Search family has two components that always work together: **SearchInput** and **TopNavSearch**.

**SearchInput** is the pill-shaped search bar itself — the input, the search icon button, the optional clear button, and the optional filter affordance. It lives in toolbars, drawers, modals, and inside the top navigation bar.

**TopNavSearch** is the specific nav-bar wrapper for the search bar. It lives in `TopNav.Global` and gives you a collapsed 48×48 icon button that springs open into the full search bar when tapped. It is not a generic container — it is a TopNav component. See the TopNavSearch section below for the full interaction spec.

> **v1 scope — search bar only.** The Open state (results dropdown) is deferred. The existing production dropdown is unchanged. Agents building products should use [Radix UI Combobox](https://www.radix-ui.com/primitives/docs/components/combobox) or Command with Pathway tokens for any dropdown they need.

## Anatomy

```
SearchInput.TouchTarget              48px height, 100% width — full touch zone
└── SearchInput.Inner (pill)         36px min-height, 64px radius, flex row
    ├── Container.LeadingIcon        24×24px icon button (search ligature)
    ├── Container.Label              flex 1 — the <input role="searchbox">
    └── Container.TrailingSlot       flex row, shrink 0
        ├── [CancelBtn]              24×24px — shown only when value.length > 0
        └── [FilterWrap]             border-left divider (showFilter=true only)
            └── FilterBtn            24×24px icon button (filter_alt ligature)
                [Badge dot]          6×6px — shown when filterBadge=true

TopNavSearch
├── CollapsedBtn                     48×48px on dark nav surface
└── ExpandedBar                      SearchInput, width animated 0→320px
```

The two-layer structure (48px touch target + 36px visual pill) ensures WCAG 2.5.5 touch compliance at every size without inflating the visual footprint. The touch target is always 48px — the visual pill is 36px.

## TopNavSearch — the nav bar search

TopNavSearch is the search component that lives inside `TopNav.Global`. It is not a generic search container — it is designed specifically for the nav bar.

<Canvas of={SearchStories.TopNavSearchStory} />

**Interaction model:**

1. **Collapsed** — renders a 48×48 icon button on the brand-blue nav surface
2. **Tap / click the icon** — the bar springs open to 320px with an intentional overshoot animation (`--motion-duration-4` · `--motion-easing-spring`)
3. **Press Escape or click the search icon inside the bar** — collapses, focus returns to the icon button
4. Input value **persists** across collapse/expand cycles

**Relationship to TopNav:** TopNavSearch is imported from `components/search/search.jsx` and placed in the `Slot.RowEnd` of `TopNav.Global`. The TopNav component manages the layout; TopNavSearch manages the search interaction. See [`components/top-nav/top-nav.jsx`](https://github.com/helloimjolopez-collab/pathway-ds/blob/main/components/top-nav/top-nav.jsx) for the integration.

## Accessibility

<table>
  <thead>
    <tr>
      <th style={{width:"200px"}}>What</th>
      <th>How</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Semantic role</td>
      <td><code>role="search"</code> on pill container, <code>role="searchbox"</code> on the <code>{'<input type="text">'}</code></td>
    </tr>
    <tr>
      <td>Search icon button</td>
      <td><code>aria-label="Search"</code> (or <code>"Collapse search"</code> when inside TopNavSearch)</td>
    </tr>
    <tr>
      <td>Clear button</td>
      <td><code>aria-label="Clear search"</code> — only present when value is non-empty, so it never announces redundantly</td>
    </tr>
    <tr>
      <td>Filter button</td>
      <td><code>aria-label="Open filters"</code> or <code>"Open filters (filters active)"</code></td>
    </tr>
    <tr>
      <td>Badge dot</td>
      <td><code>aria-hidden="true"</code> — decorative; state communicated via the filter button label change</td>
    </tr>
    <tr>
      <td>TopNavSearch</td>
      <td><code>aria-expanded</code> on the collapsed button; Escape collapses and returns focus</td>
    </tr>
    <tr>
      <td>Touch targets</td>
      <td>48×48px outer (WCAG 2.5.5 AAA); icon buttons are 24×24px visual, 44×44px effective via bar height</td>
    </tr>
    <tr>
      <td>Focus ring</td>
      <td><code>:focus-visible</code> outline, 2px solid, 2px offset, 12px radius — keyboard only, never on mouse click</td>
    </tr>
  </tbody>
</table>
