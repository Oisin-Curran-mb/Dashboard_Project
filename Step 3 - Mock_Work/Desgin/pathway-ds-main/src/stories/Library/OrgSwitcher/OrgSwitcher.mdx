# Org Switcher

The Org Switcher is the trigger that renders inside the `TopNav` component (`components/top-nav/`) and shows the active organisation. It displays the org name only — no logo or avatar is shown by default. Clicking it opens a panel for switching between orgs the user has access to.

> **v1 scope — trigger only.** The dropdown menu / panel is **out of scope for v1 and remains as it is in production until further notice.** See `org-switcher-spec.md` §0.

> **Logo display** — The org logo avatar is optional. By default (`logoUrl` absent) the trigger shows org name only. Pass `logoUrl` to render the logo image inside a 24×24 avatar frame. The church SVG placeholder is no longer shown — if there is no logo, there is no avatar.

## Catholic vs Protestant — spec §0.1

Per spec §0.1, `Container.CityName.Catholic` is **only ever rendered for Catholic organisations**. Protestant orgs render the org name alone, no pipe, no second container — even if a `cityName` string is supplied.

The rule is enforced in code by the `orgType` discriminant:

```jsx
const showCityName = orgType === "catholic" && Boolean(cityName);
```

## Mobile

On viewports below 768px (or when `mobile={true}` is forced), the trigger compacts to a fixed 108px-wide pill. The label container is a fixed 50px wide and the full org name truncates inside it with ellipsis. **No abbreviation.** Typography is `Label/Button/S` (14px) — same as desktop.

## Accessibility

- Trigger is a real `<button>` with `aria-haspopup="true"` and `aria-expanded` reflecting the open state
- `aria-label` always carries the full org name, even when the visible label is truncated
- For Catholic orgs, the city name is included in the aria-label (`"Current organisation: Sacred Heart Church-ITD, Knoxville. Activate to switch."`)
- Outer touch target is 48×48 (WCAG 2.5.5)
- Disabled state (single-org users) sets `aria-disabled` via the native `disabled` attribute
- Reduced motion: chevron rotation drops to instant when `prefers-reduced-motion: reduce` is set at the shell level

See spec §11 for the full accessibility section including ARIA markup examples, screen reader output strings, and WCAG contrast ratios.
